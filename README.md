# Cinema Lisboa

Static website showing movie showtimes for cinemas in the Lisbon area. Scraped hourly from [cinecartaz.publico.pt](https://cinecartaz.publico.pt/), enriched with IMDb/RT ratings from [OMDb](https://www.omdbapi.com/), and deployed to GitHub Pages.

**Coverage:** NOS, UCI, Cinema City, Castello Lopes, and independents in the Lisboa + Grande Lisboa regions.

## How it works

```
GitHub Actions (hourly cron)
  -> Python scraper fetches cinecartaz homepage + movie pages
  -> Enriches with OMDb ratings + original language
  -> Commits site/data/showtimes.json
  -> GitHub Pages serves site/ with new data
```

Cinemas are sorted by distance from **Paço de Arcos** by default. Users can opt into browser geolocation via the 📍 button.

## Limitations (honest)

- **Cinecartaz only publishes 3 days of showings** (today + 2). Longer ranges need direct chain scraping.
- **VO/VP inference is imperfect.** Adult films rarely have explicit `(VO)` tags on cinecartaz — we assume untagged = VO by convention. Sessions show a `~` marker when the VO classification is inferred rather than explicit. Always verify on the cinema's own site before traveling.
- **Only language is shown** (not "VO dubbed in English"). Language comes from OMDb's `Language` field for the movie; it represents the *original* language of the film.

## Local development

```bash
python3 -m venv .venv
.venv/bin/pip install -r scraper/requirements.txt

# Quick scraper test (no OMDb, 3 movies):
.venv/bin/python scraper/main.py --output-dir site/data --max-movies 3 --no-enrich

# Full scrape with ratings:
OMDB_API_KEY=your_key .venv/bin/python scraper/main.py --output-dir site/data

# Serve the static site:
python3 -m http.server -d site 8000
# Open http://localhost:8000/
```

## Deployment setup

1. Create a public GitHub repo.
2. Get a free OMDb API key at <https://www.omdbapi.com/apikey.aspx>.
3. In **Settings → Secrets and variables → Actions**, add `OMDB_API_KEY`.
4. In **Settings → Pages**, set source to **"GitHub Actions"**.
5. Push this code. The workflow auto-runs on push, then hourly.
6. Site is live at `https://<username>.github.io/<repo>/`.

## Files

| Path | Purpose |
|---|---|
| `scraper/main.py` | Orchestrates the scrape + enrichment + write |
| `scraper/cinecartaz.py` | Fetches and parses cinecartaz pages |
| `scraper/omdb.py` | Fetches ratings + language from OMDb |
| `scraper/cinemas_meta.py` | Hardcoded coords + chain mapping per cinema ID |
| `scraper/models.py` | Pydantic schema for `showtimes.json` |
| `site/index.html` | Static page |
| `site/style.css` | Styles (dark mode via `prefers-color-scheme`) |
| `site/app.js` | Client-side filtering, distance sorting, rendering |
| `site/data/showtimes.json` | Generated by the scraper, consumed by the frontend |
| `.github/workflows/update.yml` | Hourly cron + Pages deploy |

## Adding a new cinema

When cinecartaz adds a new cinema in the Lisboa region, it appears automatically in the output with inferred chain from the name. To improve accuracy (add coordinates for distance sorting, fix chain classification), edit `scraper/cinemas_meta.py` and add an entry to `CINEMA_META` keyed by the cinecartaz cinema ID.

Find the ID by fetching the homepage and looking for `window.Rooms["Lisboa"].locals.push({ id: ..., name: ... })`.
